==============================================================================
*project.txt*                   A dynamic project management solution for Neovim
                                                          *project* *project-nvim*


    `project.nvim` is a Neovim plugin written in Lua that,
    under configurable conditions, automatically sets
    the user's `cwd` to the current project root
    and also allows users to manage, access and selectively include
    their projects in a history.

    This plugin allows you to navigate through projects,
    "bookmark" and/or discard them, according to your needs.

    This was originally forked from `ahmedkhalf/project.nvim`.
    Ever since I've decided to extend it and address issues.

            This plugin will be maintained and extended ~
            for the foreseeable future. ~

1. project.setup                                                 *project.setup*
                                                                *project.config*


    To enable the plugin you must call `setup()`.

    Parameters ~

    • `{options}`  (|ProjectOpts|)
                 The options table.
                 The `{options}` parameter is not required, but if you're
                 interested in the default config options, see |ProjectOpts|
                 for all the defaults

    Example ~

    To use the defaults:
>lua
    require('project').setup()
<

1.1. `ProjectOpts`                                                   *ProjectOpts*
                                                         *project.setup-options*


    • {before_attach}           (`nil|function`)

                                will the plugin not execute.
                                It's a table with two fields:
                                `ft` (|'filetype'|) and `bt` (|'buftype'|),

                                CREDITS TO @Zeioth! ~

                                project.nvim's `README.md` ~

                                e.g. `{ '~/.cargo/*', ... }`

                                If `true` your root directory won't be changed

2. Pattern Matching                                   *project.pattern-matching*


    `project.nvim` comes with a `vim-rooter`-inspired pattern matching
    expression engine to give you better handling of your projects.

    Examples ~

    To specify the root is a certain directory, prefix it with `=`
>lua
    patterns = { '=src' }
<

    To specify the root has a certain directory or file (which may be a glob),
    just add it to the pattern list
>lua
    patterns = { '.git', '.github', '*.sln', 'build/env.sh' }
<

    To specify the root has a certain directory as an ancestor
    (for excluding directories), prefix it with `^`
>lua
    patterns = { '^fixtures' }
<

    To specify the root has a certain directory as its direct ancestor/parent
    (for putting working projects in a common directory), prefix it with `>`
>lua
    patterns = { '>Latex' }
<

    To exclude a pattern, prefix it with `!`
>lua
    patterns = {
      '!.git/worktrees',
      '!=extras',
      '!^fixtures',
      '!build/env.sh',
    }
<

            NOTE: Make sure to put your pattern exclusions first, ~
                  and then the patterns you DO want included ~

3. Lualine                                                     *project.lualine*


    You can add the `project.nvim` component to your statusline
    using `lualine.nvim`:

>lua
    lualine_b = {
      {
        "project",

        -- - `'short'` Only shows the basename of the project root directory (default)
        -- - `'full'` Shows the full path but without expanding the home directory
        -- - `'full_expanded'` Shows the full, expanded path
        format = 'short',

        -- Text to display when no project root is found
        -- (set to `nil` or empty string to disable)
        no_project = 'N/A',

        -- The separator
        separator = " ",

        -- Optional table of two strings set as enclosing characters.
        -- Set to `nil` to disable it
        --
        -- e.g. `enclose_pair = { '(', ')' }` ==> `(<YOUR_PROJECT>)`
        --      `enclose_pair = { '<', ']' }` ==> `<<YOUR_PROJECT>]`
        --      `enclose_pair = { nil, 'a' }` ==> `<YOUR_PROJECT>a`
        enclose_pair = nil,
      }
    }
<

6. Telescope                                                 *project.telescope*


    To enable `telescope.nvim` integration use the following code
    in your config:
>lua
    -- Lua
    require('telescope').setup(...)
    require('telescope').load_extension('projects')
<

    Example ~
>lua
    -- Lua
    require('telescope').setup({
      extensions = {
        projects = {
          layout_strategy = "horizontal",
          prompt_prefix = "󱎸  ",
          layout_config = {
            anchor = "N",
            height = 0.25,
            width = 0.6,
            prompt_position = "bottom",
          },
        },
      },
    })
<
    After that you can now call it from the command line:
>vim
    " Vimscript
    :Telescope projects
<

7. The `:Project` Command                                               *:Project*


                    The `:Project` command, if run without any arguments,
                    will open a UI window pointing to all the useful
                    operations this plugin can provide.

                    Below is a list of all the supported subcommands, and how
                    you can use them


`:Project log [clear|close|open|toggle]`                              *ProjectLog*

                    IMPORTANT ~
                    This command will not be available unless you set
                    `log.enabled = true` in your setup.

                    The `log` subcommand handles the `project.nvim`
                    log file.

                    The valid arguments are:

                    - `clear`: Clears the log file
                    - `close`: Closes the Log Window
                    - `open`: Opens the Log Window
                    - `toggle`: Toggles the Log Window

                    Useful for debugging


`:Project health`                                                  *ProjectHealth*

                    The `health` subcommand essentially runs
                    `:checkhealth project` in a single command


`:Project[!] history [clear]`                                     *ProjectHistory*

                    The `history` subcommand handles the project history.

                    If the command is called without any arguments, it will
                    toggle the `project.nvim` history file in a new tab, which
                    can be exited by pressing `q` in Normal Mode.

                    DANGER ZONE! ~
                    If called with the `clear` argument your project history
                    will be cleared. If you want to avoid a "Yes/No" prompt
                    you can call the command with a bang (`!`) to force it


`:Project fzf`                                                        *ProjectFzf*

                    The `fzf` subcommand will ONLY be available if you have
                    `fzf-lua` installed and loaded, and enabled its
                    integration in your setup.


`:Project[!] picker`                                               *ProjectPicker*

                    This subcommand works ONLY if you have `picker.nvim`
                    installed and loaded.

                    The `picker` subcommand is a dynamically enabled
                    user command that runs `project.nvim` through `picker.nvim`.
                    If a bang `!` is passed and you don't already have
                    `picker.hidden` set to `true`, then a selected project
                    will show hidden files.

                    This is an alias for `:Picker project`


`:Project snacks`                                                  *ProjectSnacks*

                    The `snacks` subcommand will ONLY work if you have
                    `snacks.nvim` installed and loaded, and enabled in your
                    setup.

                    This is an alias for:
>lua
                    require('project.extensions.snacks').pick()
<

`:Project telescope`                                            *ProjectTelescope*

                    The `telescope` subcommand will ONLY work if you have
                    `telescope.nvim` installed and loaded.

                    This is an alias for `:Telescope projects`


`:Project[!] add [</path/to/dir>[ ...]]`                              *ProjectAdd*

                    The `add` subcommand is a manual hook that prompts the user
                    for any directory through a UI prompt to be saved to your
                    project history. If your prompt is valid, your `cwd`
                    will be switched to said directory. Adding a [`!`]
                    will set the prompt to your cwd. This is particularly
                    useful if you've enabled {manual_mode} in `setup()`.

                    See |project.setup-options|


`:Project[!] config`                                               *ProjectConfig*

                    The `config` subcommand will toggle your current config
                    in a floating window, making it easier to access.
                    To exit the window you can either press `q` in normal mode
                    or by runnning `:Project config` again. If called with
                    a bang `!` then the output will instead be printed as a
                    message (see |:messages|).

                    Useful for debugging


`:Project[!] delete [/path/to/first [/path/to/second [...]]]`      *ProjectDelete*

                    The `delete` subcommand is a utility to delete
                    your projects. If no arguments are given, a popup with
                    a list of your current projects will be opened.
                    If one or more arguments are passed it will expect
                    directories separated by a space. The arguments have to be
                    directories that are returned by `get_recent_projects()`
                    (see |project.get_recent_projects()|).

                    The arguments can be relative, absolute or un-expanded
                    (`~/path/to/project`). The command will attemptto parse
                    the arguments and, unless a `!` is passed to the command,
                    arguments will be verified. If passing a `!` then
                    invalid arguments will be ignored. If there is
                    a successful deletion you'll recieve a notification
                    denoting success.


`:Project[!] root`                                                   *ProjectRoot*

                    The `root` subcommand is a manual hook to set the
                    current working directory to the current file's root,
                    attempting to use any of the detection methods set
                    by the user (|ProjectOpts.detection_methods|).
                    Adding a [`!`] will make this command verbose.

                    This is particularly useful if you enabled {manual_mode}
                    in your `setup()` (see |project.setup-options|)


`:Project session`                                                *ProjectSession*

                    IMPORTANT ~
                    This subcommand requires `fd` to be installed ~
                    for it to work! ~

                    The `session` subcommand opens a custom picker, using
                    |vim.ui|, with a selection of your current session projects,
                    stored in `History.session_projects`.
                    Bear in mind this table gets filled on runtime. ~

                    If you select a session project your `cwd` will be changed
                    to what you selected. If the command is called with [`!`]
                    (`:Project! session`) then the UI will close.
                    Otherwise, another custom UI picker will appear for you
                    to select the containing files/dirs. Selecting a directory
                    will open another UI picker with its contents, and so on


`:Project[!] export [</path/to/file[.json]> [<INDENT>]]`           *ProjectExport*

                    WARNING ~
                    Use this script with caution! ~
                    You may overwrite important files if using recklessly! ~

                    The `export` subcommand allows the user to save their
                    project history in a JSON format, allowing for a
                    custom indent level if desired.

                    If the target file already exists and is not empty then
                    a confirmation prompt will appear.
>vim
                    :Project export " Will open a prompt
                    :Project export a " Output file is `a.json`
                    :Project! export b 12 " Output file is `b`, with a tab size of 12

                    :Project! export ~/.c.json " The output file will be `~/.c.json`
<

`:Project[!] import [</path/to/file[.json]>]`                      *ProjectImport*

                    The `import` subcommand allows the user to retrieve
                    their saved project history in a JSON format:
>vim
                    :Project import " Will open a prompt

                    " Will be treated as `a.json`
                    :Project export a
                    :Project import a

                    " Will be treated as `b`
                    :Project! export b
                    :Project! import b
<

8.2. `get_recent_procjects()`                     *project.get_recent_procjects()*


    This utility returns either an empty table `{}` or a string array
    `{ '/path/to/project1', ... }` containing your recent projects.

    Example ~

    You can get a list of recent projects like this:
>lua
    local recent_projects = require('project').get_recent_projects()

    -- Using `vim.notify()`
    vim.notify(vim.inspect(recent_projects))

    -- Using `vim.print()`
    vim.print(recent_projects)

8.3. `get_config()`                                         *project.get_config()*


    If `setup()` (|project.setup|) has been called, it returns a table
    containing the currently set options. Otherwise it will return `nil`.

    Example ~
>lua
    -- Lua
    local config = require('project').get_config()

    vim.notify(vim.inspect(config)) -- Using `vim.notify()`
    vim.print(config) -- Using `vim.print()`
<

------------------------------------------------------------------------------
8.4 `get_history_paths()`                            *project.get_history_paths()*


    If no valid args are passed to this function then it will return
    the following dictionary:
>lua
    -- Lua
    local get_history_paths = require('project').get_history_paths

    -- A dictionary table containing all return values below
    vim.print(get_history_paths())
    --- {
    ---     datapath = <datapath>,
    ---     projectpath = <projectpath>,
    ---     historyfile = <historyfile>,
    --- }
<
    Otherwise, if either `'datapath'`, `'projectpath'` or `'historyfile'`
    are passed, it will return the string value of said parameter:
>lua
    -- The directory where `project` sets its `datapath`
    vim.print(get_history_paths('datapath'))

    -- The directory where `project` saves the project history
    vim.print(get_history_paths('projectpath'))

    -- The path to where `project` saves its recent projects history
    vim.print(get_history_paths('historyfile'))

10. Troubleshooting                                    *project.troubleshooting*


------------------------------------------------------------------------------
10.1. History File Not Created


    If you're in a UNIX environment make sure you have `read`, `write`
    and `access` permissions (`rwx`) for the `projectpath` directory.

    - The default value is `$XDG_DATA_HOME/nvim/project_nvim`
    - See |stdpath()| for more info

    You can get the value of `projectpath` by running in the cmdline:
>vim
    :lua vim.print(require('project').get_history_paths('projectpath'))
<
    If you lack the required permissions for that directory,
    you could delete the directory (RECOMMENDED).
    Alternatively, you can change the directory permissions in your shell:
>bash
    chmod 755 <project/path>
